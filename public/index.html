演示定位

从用户 prompt、检索证据到模型输出，完整观察不确定性如何积累

用户输入的问题叫 prompt。系统会先分析 prompt 本身是否短、泛、含糊；再用 RAG 检索本地知识库或联网证据，计算证据分布是否集中；最后根据模型返回的 token 概率计算输出端不确定性。

01 Prompt 输入熵 H_prompt

衡量用户问题的信息分布和含糊程度，过短或泛化问题会提高输入风险。

02 RAG 检索熵 H_ret

衡量证据是否集中。证据分散或无证据时，回答依据不稳。

03 输出熵 H_ans

根据 token logprobs 定位模型生成时更不确定的位置。

04 综合风险 R

把输入、检索、输出、冲突、敏感度和模型先验合成处置建议。
